    0.1.1 Broken Build Configuration

    This can make the Project stop from building.

    Here is a an example of the output:

    warning: The element 'BuildConfiguration' in namespace 'https://www.unrealengine.com/BuildConfiguration' has invalid child element 'ProcessorCountMultiplier' in namespace 'https://www.unrealengine.com/BuildConfiguration'. List of possible elements expected: 'bUseUnityBuild, bCheckLicenseViolations, bUseUBTMakefiles, ArtifactDirectory, DefaultWarningLevel, bAdaptiveUnityCompilesHeaderFiles, bOmitPCDebugInfoInDevelopment, MaxParallelActions, bUseVerse, ModuleIncludePrivateWarningLevel, bBreakBuildOnLicenseViolation, bUseInlining, bRetainFramePointers, bStripSymbols, bXGENoWatchdogThread, bSupportEditAndContinue, bPublicSymbolsByDefault, StaticAnalyzer, ModuleIncludePathWarningLevel, bUseIncrementalLinking, bDeterministic, bUseShippingPhysXLibraries, StaticAnalyzerOutputType, BaseLogFileName, bAllowXGE, ClangCompileActionWeight, bCodeCoverage, bTuneDebugInfoForLLDB, ...

    How to fix this:

    Delete the "BuildConfiguration.xml" file from directory "\AppData\Roaming\Unreal Engine\UnrealBuildTool\"

    0.2.1 I have a bunch of Images instead of a Video

    This is a normal thing for 3D programs to output an Image Seqence, instead of a video, any editing software should have a way of importing that as a video. If that is not the case it can be easily converted to Video too!

    First download a program called ffmpeg this can be done via the terminal or Search via the Web.

    Install ffmpeg terminal command:

    winget install Gyan.FFmpeg

    Then open the terminal in the folder where the Image Seqence is and run this command:

    ffmpeg -f image2 -r 24 -i imagename.%04d.jpeg -vcodec libx264 -crf 18  -pix_fmt yuv420p output.mp4

    Set the framerate with:

    -r 24

    And make sure the Image name is set correct 

    imagename.%04d.jpe

    The .%04d.jpeg sets how many characters the file name contains, in this case the filename is imagename.0000.jpeg 4 characters.
